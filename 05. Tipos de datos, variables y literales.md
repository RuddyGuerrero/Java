# Tipos de Datos, Variables y Literales en Java

## 1. Introducción

En Java, toda información que el programa maneja tiene un **tipo de dato**. Los tipos determinan:
- Qué valores puede almacenar la variable
- Qué operaciones se pueden realizar sobre ella

## 2. Tipos de Datos en Java

### 2.1 Tipos primitivos

| Tipo | Tamaño | Rango | Clase envolvente | Descripción | Ejemplo |
|------|-------|-------|-----------------|-------------|---------|
| byte | 1 byte | -128 a 127 | Byte | Entero pequeño | `byte b = 10;` |
| short | 2 bytes | -32,768 a 32,767 | Short | Entero corto | `short s = 1000;` |
| int | 4 bytes | -2,147,483,648 a 2,147,483,647 | Integer | Entero | `int i = 50000;` |
| long | 8 bytes | -9,223,372,036,854,775,808 a 9,223,372,036,854,775,807 | Long | Entero largo | `long l = 100000L;` |
| float | 4 bytes | ±1.4e-45 a ±3.4028235e38 | Float | Decimal | `float f = 3.14f;` |
| double | 8 bytes | ±4.9e-324 a ±1.7976931348623157e308 | Double | Decimal grande | `double d = 3.14159;` |
| char | 2 bytes | 0 a 65,535 (Unicode) | Character | Carácter Unicode | `char c = 'A';` |
| boolean | al menos 1 bit (depende de la JVM) | true o false | Boolean | Verdadero o falso | `boolean flag = true;` |

### 2.2 Tipos de referencia
- Objetos, arreglos y cadenas (`String`).
- No tienen un tamaño fijo; dependen de la JVM.

## 3. Variables

- Una **variable** es un espacio en memoria donde se almacena un valor.
- Se declara indicando el tipo seguido del nombre:

```java
int edad;
double precio;
String nombre;
```

- Se debe **inicializar** la variable antes de usarla, por ejemplo, antes de imprimirla, para evitar errores de compilación:

```java
int edad = 25;
System.out.println("Edad: " + edad);
```

- Se puede inicializar al declarar:

```java
int edad = 25;
String nombre = "Ruddy";
```

## 4. Reglas para nombrar variables

1. Deben comenzar con letra, `$` o `_`.
2. No pueden contener espacios.
3. No pueden ser palabras reservadas de Java (`int`, `class`, `for`, etc.).
4. Sensibles a mayúsculas y minúsculas (`edad` ≠ `Edad`).
5. No tienen límite de tamaño para el nombre.
6. Deben usar la convención **CamelCase** para variables de múltiples palabras (`salarioMensual`, `nombreCompleto`).
7. Deben ser descriptivas y legibles.

**Ejemplos válidos:**
```java
int edad;
double salarioMensual;
String nombreCompleto;
```

**Ejemplos inválidos:**
```java
int 1edad;
double mi salario; // espacio no permitido
String class;       // palabra reservada
```

## 5. Literales en Java

- Son **valores constantes** que se escriben directamente en el código.
- Tipos de literales según el tipo de dato:

| Tipo de dato | Ejemplo |
|--------------|---------|
| int | `100`, `0b1010`, `0xFF` |
| long | `100000L`, `0xFFFFL` |
| float | `3.14f`, `2.71f` |
| double | `3.14159`, `2.71828` |
| char | `'A'`, `'\n'` |
| boolean | `true`, `false` |

- **Sistemas numéricos disponibles en literales enteros:**

| Sistema | Ejemplo |
|---------|---------|
| Decimal | `42` |
| Binario | `0b101010` |
| Octal | `052` |
| Hexadecimal | `0x2A` |

## 6. Explorar clases con `javap`

- Podemos usar `javap` para ver los métodos, campos y constructores de cualquier clase compilada.

Ejemplos:

```bash
javap java.lang.Long
```

- Esto mostrará todos los métodos y constructores disponibles de la clase.

## 8. Resumen

- Java tiene tipos primitivos y de referencia.
- Las variables almacenan valores y deben declararse con su tipo.
- **Siempre inicializar variables antes de usarlas** para evitar errores de compilación.
- Seguir las reglas de nomenclatura evita errores.
- Usar **CamelCase** mejora la legibilidad de nombres de variables.
- Los literales representan valores fijos usados directamente en el código.
- Los literales enteros pueden expresarse en distintos sistemas numéricos: decimal, binario, octal y hexadecimal.
- `javap` permite explorar métodos y clases de la API de Java.
