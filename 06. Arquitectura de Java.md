# Arquitectura de Java

## 1. Características de Java

- Lenguaje orientado a objetos: Facilita la organización del código usando clases y objetos, lo que mejora la reutilización y el mantenimiento.
- Simple y fácil de aprender: Su sintaxis es clara y está diseñada para evitar errores comunes, lo que lo hace accesible para principiantes.
- Seguro: Incluye mecanismos como el manejo de memoria automático y el sandboxing para evitar accesos no autorizados.
- Multihilo: Permite ejecutar varias tareas al mismo tiempo dentro de un mismo programa, mejorando el rendimiento.
- Robusto: Maneja errores de forma eficiente y reduce fallos (por ejemplo si el programa se queda sin memoria) gracias a su gestión automática de memoria y verificación estricta del código.
- Portátil: El código compilado en Java puede ejecutarse en cualquier sistema operativo que tenga una JVM.
- Independiente de la plataforma: Gracias al bytecode y la JVM, el mismo programa funciona igual en Windows, Linux, macOS, etc.
- Alto rendimiento gracias al JIT: El compilador Just-In-Time convierte partes del bytecode en código máquina optimizado durante la ejecución.
- Distribuido: Incluye herramientas para crear aplicaciones que funcionan en redes y sistemas distribuidos.

## 2. Qué es un compilador

Un compilador es un programa que traduce el código fuente completo a un lenguaje intermedio o lenguaje máquina antes de ejecutarlo.
- Detecta errores en tiempo de compilación
- Genera un archivo ejecutable o intermedio

## 3. Qué es un intérprete

Un intérprete ejecuta el programa línea por línea, traduciendo y ejecutando al mismo tiempo.
- Más lento que un compilador
- Útil para pruebas rápidas
- No genera un archivo ejecutable final

## 4. Compilador vs Intérprete

| Característica | Compilador | Intérprete |
|----------------|------------|------------|
| Traducción | Todo el programa | Línea por línea |
| Velocidad | Más rápido | Más lento |
| Archivo generado | Sí | No |
| Errores | Antes de ejecutar | Durante la ejecución |

## 5. Java como lenguaje híbrido

Java combina compilación e interpretación:
1. El compilador javac convierte .java → .class (bytecode).
2. La JVM interpreta o compila ese bytecode usando un JIT.

## 6. Proceso de ejecución en Java

.java → .class → JVM → Lenguaje máquina

## 7. Platform Independent

### Por qué Java es independiente de la plataforma

- El compilador genera bytecode independiente del sistema operativo.
- Ese bytecode siempre se ejecuta a través de la JVM.
- Cada sistema operativo tiene su propia JVM.
- Mientras exista una JVM, el mismo programa Java puede ejecutarse en cualquier lugar.

### Explicación detallada

- El bytecode es un programa intermedio y libre de errores, pero no ejecutable directamente.
- La JVM lo interpreta o lo compila con JIT a código máquina.
- La JVM no interactúa directamente con el hardware, sino a través del sistema operativo.
- Como el bytecode es igual en todas las plataformas, Java puede ejecutarse en Windows, Linux, macOS, etc.
